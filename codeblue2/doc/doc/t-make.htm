Installation

Compiling Source Code and Starting Sector

Prerequisite

The Sector/Sphere library is distributed with source code, example applications, and documentation. Currently the complete Sector software suite, including both server and client, only works on Linux. We will port the client to Windows in the near future.

The software is written in C++. GCC/G++ version 3.4 or above is required. Openssl development library is required (e.g., libssl-dev for Debian or openssl-devel for Fedora).

There are four parts in the software suite: the security server, the master server, the slaves, and the clients. These four parts can be installed on the same node or over multiple nodes. In a typical installation, The security server and the master server are on one node, while each slave runs on one node.

Compile and Install

You may either download the tarball release (current version is 1.16) or check out the most recent code from SourceForge SVN. The source code will be located under a directory called "codeblue2". When you are ready, follow the several steps below to install Sector.

  1. in ./codeblue2, do make
  2. Optional: configure key/certificate, user accounts, and slave access control list
  3. update ./master/master.conf, for server port, security server address, data directory, etc.
  4. copy the complete codeblue2 directory to all slaves and client locations.
  5. update ./slave/slave.conf, for master address, data directory, etc.
  6. start the security server, the master, and the slaves.

If you have any questions, please post them on SourceForge http://sourceforge.net/forum/forum.php?forum_id=593084.

Start the Servers

The security server is located under directory ./codeblue2/security. You can start the security server in this directory by running ./sserver. By default the security server will listen on port 5000, you may use ./sserver <port> to specify a different port.

The Sector master server is in the directory of ./codeblue2/master. you may start the master server by using command ./start_master. Again, the command must be used in the directory of ./codeblue2/master, because it needs to locate the configuration file in the same directory.

Alternatively, you may use the ./start_all and ./stop_all commands to start/stop the master and all the slaves. (They do NOT start/stop the security server.) In order to use this feature, you need to update the "slaves.list" file in this directory. Sector will read text this file and use "ssh" to start remote slaves. The public SSH key of the master node must be put in the file ~/.ssh/authorized_keys of all slave nodes before you can use these two commands. All slave nodes on this list can also be restarted automatically by the master if the slave is not responsive.

If you do not start the slaves with the ./start_all command in the master directory, you can start each slave individually with ./start_slave command under ./codeblue2/slave directory. This may be helpful for debuging purpose, since you can see all the output from the slaves.
