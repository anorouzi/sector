Installation

Understanding the Certificates

Sector uses SSL for secure communication between clients and the master, and between the master and the security server. This requires two sets of SSL key/certificate pairs before the system can start to work. The two sets of key/certificate are secuirity_node.key/security_node.cert, and master_node.key/master_node.cert.

security_node.key/security_node.cert

The key "security_node.key" must exist in the security server directory, ./codeblue2/security. The certificate security_node.cert must exist in the master server directory, ./codeblue2/master. This key/certificate pair is used for the secure communication between the security server and the master server.

master_node.key/master_node.cert

The key "master_node.key" must exist in the master server directory, ./codeblue2/master. The certificate master_node.cert must exist on ever slave node in the directory of ./codeblue2/slave and on every client in the same directory as that of the client application. This key/certificate pair is used for the secure communication between the master server and the slaves, and between the master and the clients.

In Sector Slaves and Slave do not communicate directly, neither does a client and any slave node. Such communication must be set up via the master server, and verified by the security server when necessary.

Generate and distribute the certificates

There is no keys and certificates included in the software releases. During the first make of the source code, the keys and the certificates will be automatically generated. The cerficates will also be copied to the slave directory and to the client directory. If there is no shared file system for the clusters, the slaves and the clients can be copied to each slave node and each client node, together with the certificates.

In case that the certificate need to be re-generated, there is a tool called "ssl_cert_gen" to do this. It is located in ./codeblue2/security. This tool generates the keys and certificates and copies the certificates to the proper directories.

usage: ssl_cert_gen security OR ssl_cert_gen master
